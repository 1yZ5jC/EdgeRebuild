# EdgeRebuild

EdgeRebuild是一个**Vibe Coding**的浏览器外壳项目，旨在提供旧版Edge的风味。


### 我们要做什么

构建一个基于 UWP 的双内核浏览器外壳，支持 EdgeHTML（怀旧模式）和 WebView2（现代 Chromium 内核），并提供经典 Edge（Spartan）风格界面。项目目标包括多皮肤、完整复刻经典功能（Hub 中心、阅读视图、Web 笔记等），并逐步实现现代浏览器的常用交互。

### 已经做了什么

#### 核心架构

- **双引擎抽象**：定义了 `IBrowserTab` 接口，实现了 `EdgeHtmlTab`（封装 `WebView`）和 `WebView2Tab`（封装 `WebView2`），支持异步导航、事件通知、引擎切换。
- **引擎切换**：可在运行时切换当前标签的渲染引擎，保留 URL 并重新加载。
- **标签管理**：完整的标签页系统，包括新建、关闭、拖拽排序、拖出窗口（独立窗口）、右键菜单（新建/刷新/关闭/关闭其他/关闭右侧/移动到新窗口）。
- **Hub 面板**：右侧滑出面板，包含收藏夹、历史记录、下载管理（WebView2 下载监听）。
- **地址栏与导航**：输入 URL 自动补全 `https://`，支持前进/后退/刷新。

#### 交互与菜单

- **拖拽系统**：左键拖拽可排序标签（盒子碰撞规则），向上/下拖离标签栏超过 15 像素会分离成独立窗口。
- **标签右键菜单**：左键拖拽与右键菜单互斥，避免冲突。
- **网页右键菜单**：WebView2 下自定义菜单（复制/粘贴/全选/导航/链接操作/图片操作/打印/查看源代码/检查元素）；EdgeHTML 使用系统默认右键菜单（因 API 限制放弃自定义）。
- **功能菜单**：工具栏新增“设置及其他”按钮，包含：
  - **渲染引擎切换**（EdgeHTML ↔ WebView2）
  - **缩放**（放大/缩小/重置，通过 JS 注入 `document.body.style.zoom` 实现，双引擎通用）
  - **打印**（WebView2 `ShowPrintUI` / EdgeHTML `window.print()`）
  - **查找**（WebView2 调用 `window.find()`，EdgeHTML 提示切换）
  - **查看页面源代码**（WebView2 获取 HTML 并弹窗显示，EdgeHTML 提示切换）
  - **皮肤**（占位，即将推出）
  - **更多工具**：扩展管理器、任务管理器（占位）、开发者工具（WebView2）
  - **重新打开关闭的标签**（使用堆栈记录 URL）
  - **将所有标签加入收藏**
  - **历史记录**（打开 Hub 面板）
  - **清除浏览数据**（清空历史、下载、WebView2 Cookie）
  - **设置 / 关于**（对话框）

#### 状态与稳定性

- 所有功能已编译通过并测试可用。
- 缩放通过 JavaScript 实现，兼容双引擎，避免了 UWP WebView2 缺少 `ZoomFactor` 属性的问题。
- 清除数据调用 `CookieManager.DeleteAllCookies()`，符合最新 API。
- EdgeHTML 的自定义右键菜单因 `ScriptNotify` 机制问题最终放弃，采用原生菜单，稳定可靠。

### 马上要做什么

1. **皮肤系统**：实现 Modern IE 和 Windows 10 Mobile 的界面布局，通过 XAML 模板动态切换。
2. **无痕模式**：为 WebView2 创建独立 `CoreWebView2Environment`，EdgeHTML 自动降级。
3. **扩展/插件系统**：提供 JSON 声明式广告拦截和用户脚本支持。
4. **数据持久化**：将收藏、历史、设置保存到 SQLite 或本地文件。
5. **其他经典功能**：阅读视图、Web 笔记、EPUB 阅读等。
